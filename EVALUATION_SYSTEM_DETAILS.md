# نظام تقييم الاستشاريين - التفاصيل الكاملة
## Consultant Evaluation System - Complete Details

---

## 📊 **ما يوجد في تقييم الاستشاريين؟**

### **المرحلة 1: استيراد البيانات (Import JSON)**
**الهدف:** إدخال عرض السعر من الاستشاري

**البيانات المُدخلة:**
1. **أتعاب التصميم (Design Fee):**
   - طريقة: `LUMP_SUM` (مبلغ ثابت) أو `PERCENTAGE` (نسبة من التكلفة الإنشائية) أو `MONTHLY_RATE`
   - المبلغ أو النسبة المئوية

2. **أتعاب الإشراف (Supervision Fee):**
   - طريقة: `LUMP_SUM` أو `PERCENTAGE` أو `MONTHLY_RATE`
   - المبلغ أو النسبة المئوية
   - مدة الإشراف المقترحة (بالأشهر)
   - فريق الإشراف (الأدوار والتخصصات والرواتب الشهرية)

3. **نطاق العمل (Scope Coverage):**
   - تغطية كل بند من بنود النطاق الإلزامية
   - حالة كل بند: `INCLUDED` أو `NOT_MENTIONED` أو `EXCLUDED`

---

### **المرحلة 2: مراجعة النطاق (Scope Review)**
**الهدف:** التحقق من تغطية الاستشاري لكل البنود الإلزامية

**ما يتم عرضه:**
- جدول بكل البنود الإلزامية للفئة المعمارية
- حالة التغطية من الاستشاري
- تكلفة المرجع لكل بند (من قاعدة البيانات)

**ما يتم حسابه:**
- **فجوات التصميم (Design Scope Gaps):** البنود المطلوبة والتي لم يغطيها الاستشاري
- **تكلفة الفجوات:** مجموع تكاليف البنود المفقودة

---

### **المرحلة 3: حساب التكلفة الحقيقية (Calculation Engine)**

#### **البيانات المصدرية:**
```
من المشروع:
- المساحة (BUA): بالقدم المربع
- تكلفة البناء: بالدرهم لكل قدم مربع
- مدة المشروع: بالأشهر
- الفئة المعمارية: (مثل: سكني، تجاري، إلخ)

من الاستشاري:
- أتعاب التصميم المقتبسة
- أتعاب الإشراف المقتبسة
- نطاق العمل المغطى
```

#### **الحسابات:**

**1. أتعاب التصميم الحقيقية:**
```
التكلفة الإنشائية الإجمالية = المساحة × سعر القدم

إذا كانت الطريقة = LUMP_SUM:
  أتعاب التصميم المقتبسة = المبلغ الثابت

إذا كانت الطريقة = PERCENTAGE:
  أتعاب التصميم المقتبسة = (التكلفة الإنشائية × النسبة) / 100

إذا كانت الطريقة = MONTHLY_RATE:
  أتعاب التصميم المقتبسة = المبلغ الشهري

أتعاب التصميم الحقيقية = أتعاب التصميم المقتبسة + فجوات التصميم
```

**2. أتعاب الإشراف الحقيقية:**
```
معامل تعديل المدة:
  إذا كانت مدة الإشراف المقترحة < مدة المشروع:
    معامل التعديل = مدة المشروع / مدة الإشراف المقترحة
  وإلا:
    معامل التعديل = 1

إذا كانت الطريقة = LUMP_SUM:
  أتعاب الإشراف = المبلغ الثابت × معامل التعديل

إذا كانت الطريقة = PERCENTAGE:
  أتعاب الإشراف = (التكلفة الإنشائية × النسبة / 100) × معامل التعديل

إذا كانت الطريقة = MONTHLY_RATE:
  أتعاب الإشراف = مجموع (الراتب الشهري × مدة المشروع × نسبة التخصص)
```

**3. التكلفة الحقيقية الإجمالية:**
```
التكلفة الحقيقية = أتعاب التصميم الحقيقية + أتعاب الإشراف الحقيقية
```

---

### **المرحلة 4: التقييم والترتيب (Results & Ranking)**

#### **معايير الترتيب:**
1. **يمكن ترتيبه (Can Rank):**
   - ✅ تم تقديم أتعاب الإشراف
   - ✅ تم تقديم نطاق العمل

2. **لا يمكن ترتيبه (Cannot Rank):**
   - ❌ لم يتم تقديم أتعاب الإشراف
   - ❌ حالة الاستشاري = DRAFT

#### **طريقة الترتيب:**
- **ترتيب تصاعدي** حسب التكلفة الحقيقية الإجمالية
- **الأول:** الأقل تكلفة
- **الثاني:** الثاني الأقل تكلفة
- **إلخ...**

#### **المعلومات المعروضة:**
```
لكل استشاري:
- الترتيب (1، 2، 3، إلخ)
- أتعاب التصميم المقتبسة
- فجوات التصميم وتكلفتها
- أتعاب التصميم الحقيقية
- أتعاب الإشراف المقتبسة
- أتعاب الإشراف الحقيقية
- التكلفة الحقيقية الإجمالية
- ملاحظات الحساب (الفجوات، تحذيرات المدة، إلخ)
```

## 🔄 **تدفق البيانات (Data Flow)**

```
1. المستخدم ينشئ مشروع
   ↓
2. المستخدم يضيف استشاريين
   ↓
3. الاستشاري يرفع عرض السعر (JSON)
   - أتعاب التصميم
   - أتعاب الإشراف
   - نطاق العمل
   ↓
4. المستخدم يراجع النطاق
   - يتحقق من تغطية البنود
   - يرى الفجوات
   ↓
5. محرك الحساب يعمل تلقائياً
   - يحسب التكلفة الحقيقية
   - يرتب الاستشاريين
   - يحفظ النتائج في قاعدة البيانات
   ↓
6. المستخدم يرى النتائج والترتيب
   - جدول بكل الاستشاريين
   - الترتيب حسب التكلفة
   - ملاحظات الحساب
   ↓
7. المستخدم يرى التقرير الاحترافي
   - تقرير PDF احترافي
   - يمكن تعديل الأرقام
   - يمكن اعتماد التقرير
```
